# ADR-0005 — برنامه مصرف ساختاریافته و رخدادمحور

- Status: Accepted
- Date: 2026-07-19

## Context

یک عدد اعشاری مانند `1/7` مشخص نمی‌کند مصرف واقعاً روزانه، هفتگی یا در روز خاصی انجام می‌شود. استفاده از نرخ پیوسته برای برنامه‌های غیرروزانه موجودی‌های کسری گمراه‌کننده تولید می‌کند و تاریخ رخداد بعدی را از بین می‌برد.

## Decision

Domain سه نوع برنامه را مدل می‌کند:

- daily: مقدار هر نوبت و تعداد نوبت در روز
- everyNDays: مقدار هر نوبت و فاصله روزها
- weekly: مقدار هر نوبت و روزهای هفته

موجودی بر اساس رخدادهای کامل‌شده کم می‌شود. میانگین روزانه فقط برای سازگاری schema قبلی نگهداری می‌شود و مبنای محاسبه برنامه‌های غیرروزانه نیست.

تغییر برنامه یک baseline جدید با موجودی تخمینی فعلی ایجاد می‌کند تا گذشته با برنامه جدید بازنویسی نشود.

ذخیره مشخصات برنامه جدید و رویداد baseline باید در یک transaction دیتابیس انجام شود. اعلان فقط پس از commit موفق دوباره زمان‌بندی می‌شود؛ شکست سیستم اعلان نباید transaction داده را rollback کند.

## Consequences

### مثبت

- کاربر دیگر محاسبه اعشاری دستی انجام نمی‌دهد.
- موجودی هفتگی و یک‌روزدرمیان بین رخدادها ثابت می‌ماند.
- اعلان و تاریخ پایان بر مبنای رخداد واقعی برنامه محاسبه می‌شوند.
- migration نسخه ۱ داده موجود را بدون حذف حفظ می‌کند.
- تغییر برنامه یک مرز تاریخی صریح و قابل ممیزی ایجاد می‌کند.

### منفی

- زمان دقیق نوبت‌ها در این نسخه جداگانه ذخیره نمی‌شود؛ برنامه روزانه رخدادها را از زمان baseline با فاصله مساوی توزیع می‌کند.
- تغییر برنامه نیازمند رویداد baseline جدید است.
- schema و تست migration پیچیده‌تر می‌شوند.
