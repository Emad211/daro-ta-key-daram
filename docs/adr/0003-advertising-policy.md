# ADR-0003 — درآمد فقط از تبلیغات با AdService مستقل

- Status: Accepted
- Date: 2026-07-18

## Context

مدل درآمد محصول بر تبلیغات است، اما اتصال مستقیم UI به یک SDK خاص تغییر provider و تست را دشوار می‌کند. همچنین محیط سلامت نیازمند محدودیت نمایش است.

## Decision

تمام SDKهای تبلیغاتی پشت `AdService` قرار می‌گیرند. سیاست frequency cap بخشی از domain محصول است، نه تصمیم پراکنده صفحه‌ها.

## Consequences

- تعویض Adivery و Tapsell آسان‌تر می‌شود.
- تست UI بدون SDK تبلیغ ممکن است.
- برای هر provider یک adapter جدا لازم است.
- اطلاعات پزشکی اجازه عبور از مرز `AdService` را ندارند.
