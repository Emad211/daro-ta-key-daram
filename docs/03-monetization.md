# 03 — مدل درآمد و سیاست تبلیغات

## مدل کسب‌وکار قطعی

- اپ برای کاربر رایگان است.
- اشتراک، خرید درون‌برنامه‌ای و فروش داده وجود ندارد.
- کل درآمد از تبلیغات داخل اپ تأمین می‌شود.

## انتخاب شبکه

### گزینه اول: Adivery

دلیل انتخاب اولیه:

- پلاگین Flutter رسمی و به‌روزشده
- تمرکز روی بازار ایران
- پشتیبانی از banner، interstitial، rewarded و app-open

### گزینه دوم: Tapsell Plus

به‌عنوان fallback یا mediation، پس از پایدارشدن نسخه اول بررسی می‌شود. هر دو شبکه پشت قرارداد `AdService` قرار می‌گیرند تا تغییر provider منطق محصول را آلوده نکند.

## جایگاه‌های MVP

### 1. Banner داشبورد

- در پایین صفحه فهرست داروها
- فقط پس از ثبت حداقل یک دارو
- نباید دکمه افزودن یا اطلاعات وضعیت را بپوشاند
- شکست بارگذاری نباید فضای خالی بزرگ ایجاد کند

### 2. Interstitial محدود

تنها پس از یک اقدام کامل‌شده و غیرحساس، مانند به‌روزرسانی موفق موجودی.

قواعد اولیه frequency cap:

- پس از اولین استفاده نمایش داده نشود.
- حداقل ۴ اقدام معنادار بین دو تبلیغ
- حداقل ۱۰ دقیقه بین دو تبلیغ
- حداکثر ۳ تبلیغ تمام‌صفحه در روز
- هنگام وضعیت بحرانی یا مشاهده داروی تمام‌شده نمایش داده نشود

### 3. Rewarded در آینده

فقط برای قابلیت‌های اختیاری و غیرضروری، مانند قالب‌های ظاهری یا گزارش پیشرفته. قابلیت اصلی محاسبه، هشدار و ثبت موجودی هرگز پشت rewarded ad قفل نمی‌شود.

## موارد ممنوع

- تبلیغ روی onboarding
- تبلیغ قبل از نمایش هشدار اتمام دارو
- interstitial هنگام بازشدن فرم یا قبل از ذخیره
- تبلیغ با ظاهر دکمه پزشکی یا هشدار سیستم
- app-open ad در MVP
- ارسال نام دارو، دوز یا بیماری به شبکه تبلیغاتی

## شاخص‌های درآمدی و محصول

### محصول

- DAU و MAU
- D1 / D7 / D30 retention
- تعداد داروهای فعال به ازای کاربر
- تعداد به‌روزرسانی موجودی در هفته
- نرخ تکمیل افزودن دارو

### تبلیغات

- Ad requests
- Fill rate
- Impressions per session
- eCPM
- Revenue per DAU
- خطای بارگذاری تبلیغ
- اثر تبلیغات بر retention و session abandonment


## Config

شناسه‌ها از `--dart-define` یا فایل config خارج از git تزریق می‌شوند:

```bash
flutter run \
  --dart-define=AD_PROVIDER=adivery \
  --dart-define=ADIVERY_APP_ID=... \
  --dart-define=ADIVERY_BANNER_PLACEMENT_ID=...
```

## اصل اقتصادی

افزایش درآمد فقط با افزایش تعداد نمایش تبلیغ دنبال نمی‌شود. ترتیب اولویت:

1. حفظ کاربر و استفاده تکرارشونده
2. افزایش DAU و session frequency
3. افزایش fill rate و کیفیت placement
4. سپس بهینه‌سازی تعداد impression
