# 08 — مدل محاسبه موجودی دارو

## هدف

این سند مدل ریاضی مورد استفاده برای تخمین موجودی، زمان پایان و وضعیت هشدار را تعریف می‌کند. این مدل یک **تخمین خطی بر اساس اطلاعات واردشده توسط کاربر** است و مصرف واقعی، فراموش‌شدن دوز یا تغییر دستور پزشک را مشاهده نمی‌کند.

## ورودی‌ها

برای هر دارو:

- `stockAtRecord`: موجودی در زمان ثبت مبنا
- `unitsPerDay`: مقدار مصرف ثبت‌شده در هر ۲۴ ساعت
- `inventoryRecordedAt`: زمان مؤثرشدن موجودی مبنا
- `alertLeadDays`: فاصله هشدار پیش از پایان

قواعد ورودی:

- موجودی باید محدود و نامنفی باشد.
- مصرف روزانه باید محدود و بزرگ‌تر از صفر باشد.
- فاصله هشدار باید بین صفر تا ۳۶۵ روز باشد.
- پوشش محاسبه‌شده باید به یک `DateTime` قابل نمایش تبدیل شود؛ پوشش نامحدود یا خارج از بازه زمانی پشتیبانی‌شده رد می‌شود.

## مدل پیوسته

```text
elapsedDays = max(0, effectiveNow - inventoryRecordedAt) / 24 hours
consumedUnits = elapsedDays × unitsPerDay
remainingUnits = max(0, stockAtRecord - consumedUnits)
remainingDays = remainingUnits / unitsPerDay
coverageDays = stockAtRecord / unitsPerDay
```

محاسبه زمان با دقت میکروثانیه انجام می‌شود؛ گردکردن برای نمایش، مسئولیت UI است و وارد Domain نمی‌شود.

## تاریخ‌ها

```text
depletionAt = inventoryRecordedAt + coverageDays
reorderCandidate = depletionAt - alertLeadDays
reorderAt = max(inventoryRecordedAt, reorderCandidate)
```

بنابراین `reorderAt` هیچ‌گاه قبل از ثبت موجودی یا بعد از زمان پایان قرار نمی‌گیرد.

## وضعیت موجودی

ترتیب اولویت:

1. `depleted`: زمان پایان رسیده یا موجودی صفر است.
2. `critical`: سه روز یا کمتر باقی مانده است.
3. `warning`: مقدار باقی‌مانده داخل فاصله هشدار انتخابی کاربر است.
4. `safe`: سایر حالت‌ها.

مرز سه‌روزه یک ثابت Domain است و باید فقط با تصمیم محصول و تغییر تست‌ها عوض شود.

## Invariantهای قطعی

برای هر snapshot معتبر:

- `estimatedRemainingUnits >= 0`
- `exactRemainingDays >= 0`
- هر دو مقدار محدود و قابل محاسبه هستند.
- با جلو رفتن زمان، موجودی و روزهای باقی‌مانده افزایش پیدا نمی‌کنند.
- `fullRemainingDays == floor(exactRemainingDays)`
- در `depletionAt` و پس از آن، موجودی و روز باقی‌مانده صفر است.
- وقتی ساعت دستگاه قبل از زمان ثبت قرار می‌گیرد، هیچ مصرفی از موجودی کم نمی‌شود.
- `inventoryRecordedAt <= reorderAt <= depletionAt`

## دقت و نمایش

Domain مقادیر خام را حفظ می‌کند. UI می‌تواند برای خوانایی عدد را قالب‌بندی کند، اما نباید مقدار گرد‌شده را دوباره به منطق محاسبه یا دیتابیس برگرداند.

برای مقادیر مثبت بسیار کوچک، UI باید در آینده به‌جای نمایش گمراه‌کننده `0` از عبارتی مانند «کمتر از ۰٫۱ واحد» استفاده کند.

## محدودیت پزشکی

- مدل فرض می‌کند نرخ مصرف در بازه ثابت است.
- مدل پایبندی بیمار را اندازه‌گیری نمی‌کند.
- تاریخ پایان تقریبی است.
- برنامه دوز را پیشنهاد، اصلاح یا تأیید نمی‌کند.
- تغییر `unitsPerDay` فقط باید بازتاب دستور پزشک یا داروساز باشد.
